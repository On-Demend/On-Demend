Commands - Linux¶

📂 1. File & Directory • 리눅스 구조 이해¶

리눅스를 처음 접하면 윈도우와 다른 파일 시스템 구조 때문에 어려울 수 있습니다.

디렉토리는 폴더를 의미합니다.

/ (루트 : 최상위 디렉토리)¶

  • 리눅스 시스템의 뿌리이자 모든 것의 시작점
  • 윈도우와의 차이: 윈도우 운영체제의 C:\ 드라이브와 비슷하지만, 리눅스에는 D:\, E:\ 같은 다른 뿌리가 없습니다. 모든 폴더와 장치는 무조건 / 아래에 있습니다.
  • 특징: 모든 경로는 반드시 / 로 시작하거나 /를 거쳐서 연결됩니다.
    • 예시 : /cloud/kmit2023/worldskills

(윈도우의 디렉토리 구조도 헷갈린다면 아무 선배에게 물어볼 것)

. (작업 디렉토리)와 .. (상위 디렉토리)¶

  • . (점 하나 - 작업 디렉토리): "지금 내가 있는 곳"을 의미 (현재 위치)
    • 작업 디렉토리 = 현재 디렉토리로 이해하면 됩니다.
  • .. (점 둘 - 상위 디렉토리): "내 바로 위 부모 폴더(상위 폴더)"를 의미 (뒤로 가기)

문제 : 내 작업 디렉토리가 /opt/app/kmit/ 일 때 /opt/app 을 나타내는 것은? (정답 : ..)¶

~ (홈 디렉토리)¶

  • 리눅스에도 각각의 사용자가 있습니다.
  • 로그인한 사용자(본인)에게 할당된 디렉토리도 별도로 존재합니다. 이것을 홈 디렉토리라고 합니다.
  • 윈도우의 내 문서나 C:\Users\이름 폴더와 똑같다고 보면 됩니다.
  • 홈 디렉토리를 나타내는 경로는 /home/<사용자 명> 입니다.
  • 현재 로그인 된 사용자의 홈 디렉토리를 나타낼 때에는 ~ 라고 합니다. (사용자 이름이 world라면 /home/world와 ~은 같은 곳을 나타냄)

절대 경로 vs 상대 경로¶

절대 경로 (Absolute Path):¶

  • 현재 자신이 어떤 경로에 있는 변하지 않는 경로입니다.
  • 반드시 /로 시작합니다.
  • 예: /var/www/html/index.html

상대 경로 (Relative Path):¶

  • 내 현재 위치를 기준으로 말하는 주소 입니다. . .. 을 사용하여 나타내기도 합니다.
  • /로 시작하지 않습니다. 내가 위치를 옮기면 경로도 달라집니다.
  • 예: ../images/logo.png (한 칸 위로 가서 images 폴더 안의 logo.png)

✅ 2. 기본 명령¶

일단 이 명령어들만 외워도 벌써 반이나 도달했습니다. 대회 나갈 때 쯤이면 손이 기억합니다.

명령어 의미 상세 설명
pwd Print Working Directory 작업 디렉토리 출력 / 현재 위치한 전체 경로를 보여줍니다.
ls List 현재 디렉토리에 뭐가 있는지 목록을 보여줍니다.
ls -al List All Long 숨겨진 파일까지 포함해서 아주 자세하게(권한, 날짜 등) 보여줍니다.
cd [경로] Change Directory 디렉토리를 이동합니다. (예: cd /etc 또는 cd ~)
cd - Back 바로 직전에 있었던 디렉토리로 되돌아갑니다.
clear Clear 지저분해진 터미널 화면을 깨끗하게 비웁니다.
exit Exit 현재 접속 중인 터미널 세션을 종료합니다.

🛠️ 3. 파일 및 디렉토리 관리¶

서버 설정을 하거나 파일을 옮길 때 필수로 사용하는 명령어들입니다.

클라우드 하면서 많이 쓰게 될 명령어 모음입니다. (어렵지 않음)

명령어 상세 설명
mkdir [이름] 디렉토리를 만듭니다. -p를 쓰면 하위 디렉토리까지 한 번에 만듭니다. (예: mkdir -p a/b/c)
touch [파일명] 내용이 없는 빈 파일을 만듭니다.
cp [원본] [대상] 파일이나 디렉토리를 복사합니다. 디렉토리까지 복사 시에는 반드시 -r 옵션이 필요합니다.
mv [원본] [대상] 파일/디렉토리를 옮기거나 이름을 변경할 때도 사용합니다.
rm [대상] 파일을 삭제합니다. -r은 디렉토리까지 삭제 가능하며, -f는 강제 삭제 옵션입니다.
cat [파일명] 파일 안에 내용이 무엇인지 터미널에서 바로 출력합니다.
tail -f [파일명] 파일의 마지막 부분을 실시간으로 보여줍니다. 서버 에러 로그 확인 시 필수입니다.
vim [파일명] 리눅스의 메모장(편집기)입니다. 파일을 수정할 때 사용합니다. (다른 페이지에서 자세히 설명할 예정)

⚙️ 4. 시스템 관리 및 권한¶

클라우드 환경에서 서비스를 띄우거나 관리 권한이 필요할 때 사용합니다.

명령어 상세 설명
sudo [명령어] Super User DO. 일반 사용자가 할 수 없는 관리자 작업을 수행합니다.
chmod [숫자] [파일] 파일의 권한(읽기/쓰기/실행)을 설정합니다. (예: chmod 755 스크립트.sh)
chown [유저]:[그룹] 파일의 주인(소유자)이 누구인지 바꿉니다.
systemctl start [서비스] Nginx나 Docker 같은 프로그램을 실행시킵니다. (stop, restart, status 등)
systemctl enable [서비스] 서버를 껐다 켜도 이 프로그램이 자동으로 실행되게 등록합니다.
df -h 디스크 용량이 얼마나 남았는지 '사람이 보기 편하게(h)' 보여줍니다.
free -h 현재 서버의 메모리(RAM) 사용량을 보여줍니다.

🌐 5. 네트워크 및 패키지¶

인터넷 연결을 확인하거나 새로운 프로그램을 설치할 때 사용합니다.

명령어 상세 설명
sudo apt update 설치 가능한 프로그램 목록을 최신판으로 업데이트합니다. (Debian 계열)
sudo apt install -y [명칭] 새로운 프로그램을 설치합니다. -y는 확인 질문을 생략합니다. (Debian 계열)
sudo dnf update 설치 가능한 프로그램 목록을 최신판으로 업데이트합니다. (RedHat 계열)
sudo dnf install -y [명칭] 새로운 프로그램을 설치합니다. -y는 확인 질문을 생략합니다. (RedHat 계열)
ip a 이 리눅스 시스템의 IP 주소가 무엇인지 확인합니다.
ping [주소] 해당 IP 주소와 통신이 잘 되는지 확인합니다.
curl [URL] 웹 사이트의 상태를 확인합니다. (매우 중요!!)
ss -tuln 현재 리눅스에서 어떤 포트가 열려 있는지 확인합니다.

🔍 6. 기타¶

명령어 상세 설명
history 지금까지 입력했던 명령어들의 기록을 보여줍니다.
grep [단어] [파일] 파일 내용 중에서 특정 단어가 들어간 줄만 골라냅니다.
[명령어] | grep [단어] 파이프라는 것으로, 앞 결과물에서 특정 단어만 필터링합니다.
tar -xvzf [파일명] .tar.gz 형식으로 압축된 파일을 풉니다. zip과 비슷한 형태입니다.
