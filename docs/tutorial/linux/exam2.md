### 1. 디렉토리

**1-2.** 현재 위치에서 한 단계 위가 아닌, '직전에 머물렀던 디렉토리'로 즉시 돌아가고 싶을 때 사용하는 `cd` 명령어의 인자(Argument)는 무엇인가요?

**1-3.** `mkdir` 명령어를 사용하여 `cloud/linux/exam` 처럼 하위 디렉토리까지 한 번에 생성하고 싶을 때 사용하는 옵션은 무엇인가요?

**1-4.** 현재 작업 디렉토리를 알 수 없는 상태에서 상대 경로만을 이용하여 디렉토리를 이동하는 명령어는 무엇인가요?

### 2. 파일 권한 관리
**2-1.** 관리자(root)가 아닌 일반 사용자가 시스템 설정 파일을 수정하기 위해 일시적으로 관리자 권한을 획득할 때 명령어 앞에 붙이는 키워드는 무엇인가요?

**2-2.** 파일 및 디렉토리의 권한은 읽기 권한, 쓰기 권한, 실행 권한 이렇게 3가지 입니다. 각각 무엇인지 설명해보세요.

- **2-2-1**. 모든 사용자가 `awscli`라는 파일을 작성/수정하거나 실행하지 못하도록 하고 싶습니다. 어떤 명령어를 입력해야할까요?

**2-3.** 파일의 권한이 `rwxr-xr-x`인 경우, 이 권한이 무엇을 의미하는지 설명해보세요.

**2-4.** 비어 있지 않은 디렉토리를 강제로, 그리고 하위 내용까지 모두 삭제하고 싶을 때 사용하는 `rm` 명령어의 옵션 조합은 무엇인가요? (※ 주의해서 사용해야 함)

- **2-4-1(선택).** 리눅스 운영체제 시스템은 오직 `/` 부터 시작합니다. 그렇다면 리눅스 시스템 자체를 지우는 명령어는 무엇인가요?

### 3. 텍스트 처리/로그 분석
**3-1.** 리눅스 서버의 로그 파일(예: `server.log`)의 내용이 너무 많을 때, 파일의 마지막 20줄만 출력해서 확인하는 명령어와 옵션은 무엇인가요?

**3-2.** 로그 파일에 새로운 내용이 추가될 때마다 화면에 실시간으로 출력되도록 유지하는 `tail` 명령어의 옵션은 무엇인가요?

**3-3.** `cat access.log` 명령의 결과물 중에서 "404"라는 단어가 포함된 줄만 골라서 보고 싶을 때, `cat`과 함께 사용하는 명령어와 기호(`|`)의 조합은 무엇인가요?

**3-4.** 명령어의 실행 결과를 화면에 출력하는 대신 `result.txt`라는 파일로 저장(덮어쓰기)하고 싶을 때 사용하는 기호는 무엇인가요?

### 4. 네트워킹
**4-1.** 현재 내 클라우드 서버에 설정된 IP 주소를 확인하는 명령어는 무엇인가요? (단, `ifconfig`가 설치되어 있지 않은 환경 기준)

**4-3.** 터미널 환경에서 웹 서버에 접속하여 HTML 소스를 가져오거나 API 응답을 확인하기 위해 사용하는 명령어는 무엇인가요? (주로 `c`로 시작함)

**4-4.** 클라우드 서버의 특정 포트(예: 80, 443)가 열려 있는지 확인하기 위해 사용하는 네트워크 상태 확인 명령어는 무엇인가요?

### 5. 패키지 설치 및 관리 (Package Management)
!!! info "Linux의 계열별 설치 명령어"

    Debian 계열 : apt(Advanced Package Tool) 라는 패키지 관리자 사용

    - 배포판 종류 : Debian, Ubuntu, Kali Linux, Linux Mint 등

    Red Hat 계열 : dnf(Dandified YUM) 라는 패키지 관리자 사용, 이전에는 yum(Yellowdog Updater Modified) 라는 패키지 관리자를 사용했으나 현재도 호환

    - 배포판 종류 : Red Hat Enterprise Linux, CentOS, Rocky Linux, AlmaLinux, Fedora, 그리고 AWS에서 쓸 **Amazon Linux** 등
**5-1.** Ubuntu 리눅스에서 새로운 소프트웨어를 설치하기 전, 최신 패키지 목록으로 업데이트하는 명령어는 무엇인가요?

**5-2.** `apache`라는 웹 서버 패키지를 설치하기 위해 사용하는 전체 명령어는 무엇인가요? (관리자 권한 포함)
