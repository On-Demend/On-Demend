# Linux 기초 명령어 시험 3

클라우드 인프라 운영하는데 가장 기본적인 명령어들 입니다.

이어지는 문제이므로 끝까지 풀어보세요.

문제가 어렵다면 `Commands`에서 참고 및 검색은 해도 좋습니다. 다만 실력 향상을 위해 인공지능 사용은 자제해주세요. 

VMWare 가상머신 등을 사용하여 Ubuntu에서 작업하세요.

---

1. 현재 서버의 운영체제(OS)가 Ubuntu인지 CentOS인지 확인하기 위해 커널 및 OS 정보를 상세히 출력하세요.
2. 현재 접속한 사용자 계정의 이름을 확인하세요.
3. 시스템이 마지막으로 부팅된 후 현재까지 가동된 시간(Uptime)을 확인하세요.
4. 서버의 현재 프라이빗 IP 주소를 확인하는 명령어를 작성하세요.
5. 최상위 루트(/) 디렉토리 아래에 `cloud_project`라는 디렉토리를 생성하세요. (권한 오류 발생 시 해결 방법 포함)
6. 생성한 `cloud_project` 디렉토리로 이동하세요.
7. 해당 디렉토리 안에 `web_log`와 `backup`이라는 두 개의 디렉토리를 한 번의 명령으로 생성하세요.
8. 현재 디렉토리의 소유자를 `ubuntu` 사용자와 `ubuntu` 그룹으로 변경하세요. (재귀적 변경 옵션 사용)
9. `web_log` 디렉토리의 권한을 '소유자는 모든 권한, 그룹과 나머지는 읽기 및 실행만' 가능하도록 숫자로 설정하세요.
10. `web_log` 디렉토리 안에 `access.log`라는 빈 파일을 생성하세요.
11. 리눅스 표준 편집기(vi 또는 nano)를 사용하여 `access.log` 파일에 "Server Started at 2026-02-25"라는 문구를 입력하고 저장하세요.
12. `access.log` 파일의 내용을 터미널 화면에 출력하여 확인하세요.
13. 기존 `access.log` 파일의 내용을 유지하면서, 그 뒤에 "New connection from 192.168.1.10"이라는 문장을 추가(Append)하세요.
14. 새로운 소프트웨어를 설치하기 전, 시스템의 패키지 리스트를 최신 상태로 업데이트하세요.
15. 웹 서버 소프트웨어인 `nginx`를 관리자 권한으로 설치하세요.
16. 설치된 `nginx` 서비스가 현재 실행 중(active)인지 상태를 확인하세요.
17. 서버가 재부팅되어도 `nginx` 서비스가 자동으로 시작되도록 설정하세요.
18. 현재 서버에서 80번 포트(HTTP)가 정상적으로 리스닝(Listening) 중인지 확인하세요.
19. 서버에서 실행 중인 전체 프로세스 목록을 페이지 단위로 끊어서 확인하세요.
20. `nginx`와 관련된 프로세스만 골라내어 해당 프로세스의 ID(PID)를 확인하세요.
21. 외부(Google 등)와 네트워크 통신이 원활한지 확인하기 위해 패킷을 4번만 보내보세요.
22. `access.log` 파일의 내용이 실시간으로 추가되는 것을 감시(Monitoring)하는 명령어를 실행하세요.
23. `access.log` 파일 내에서 "192.168"이 포함된 행만 필터링하여 출력하세요.
24. `web_log` 디렉토리 전체를 `web_archive.tar.gz`라는 이름으로 압축하여 `backup` 디렉토리 안에 저장하세요.
25. `backup` 디렉토리에 있는 파일 목록을 용량(KB, MB 등)과 함께 상세하게 출력해서 압축이 잘 되었는지 확인하세요.
